# Report Structure

Reports lead with an executive score summary, measured category scores, pages checked, issue priority, and the most important fixes. Routed section pages then expose SEO, technical, crawlability, internal-link, performance, passive-security, and page-level evidence.

Every finding presents plain-language impact and remediation before optional technical evidence. Performance values are labelled as audit-time HTTP response and HTML payload observations, not Lighthouse or field Core Web Vitals. Passive security sections state that checks are non-invasive.

PDF export uses the same stored audit result and is available according to the active plan. Print styles use A4 margins and avoid splitting cards, tables, and rows where the browser print engine supports it. JSON and CSV exports remain available for technical workflows.
