# Public Artifact Review

Run this review before a public release.

1. Run the craftsmanship, copy, placeholder, privacy, SEO, and security smoke tests.
2. Inspect public pages for developer notes, infrastructure names, local paths, fake statistics, unlabeled samples, and generator attribution.
3. Confirm unfinished routes return the product 404 or remain behind an explicit feature flag.
4. Verify example reports are labeled in the same panel as sample values.
5. Review customer errors separately from protected administrator diagnostics.
6. Check homepage, authentication, audit progress, report, history, settings, blog, legal, and 404 routes at mobile, tablet, and desktop widths.
7. Test light and dark themes, keyboard navigation, reduced motion, and browser zoom.

Historical Git content is not rewritten automatically. Report any personal path or credential found in history to the repository owner for a separate remediation decision.
